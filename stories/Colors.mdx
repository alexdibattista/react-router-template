# Colors & Theme

Our color system is designed to create consistent, accessible, and visually cohesive interfaces using OKLCH color values for better color perception and contrast. Each color has a specific purpose and semantic meaning within the application.

## Color System

Our colors are defined using OKLCH (Lightness, Chroma, Hue) color space which provides more perceptually uniform colors across different devices and better accessibility. We use CSS custom properties to maintain a consistent theme throughout the application.

## Theme Modes

Our application supports both light and dark modes, with colors automatically adjusting based on the selected theme.

### Light Mode (Default)

The light theme uses lighter backgrounds with darker text for standard reading conditions.

```jsx
<div className="bg-background text-foreground p-4 rounded-lg">
  <h3 className="text-lg font-medium">Light Mode Component</h3>
  <p className="text-muted-foreground">Content in light mode</p>
  <button className="bg-primary text-primary-foreground px-4 py-2 rounded-md">Primary Action</button>
</div>
```

### Dark Mode

The dark theme uses darker backgrounds with lighter text to reduce eye strain in low-light environments. The `.dark` class is applied to the root element to activate dark mode.

```jsx
<div className="dark">
  <div className="bg-background text-foreground p-4 rounded-lg">
    <h3 className="text-lg font-medium">Dark Mode Component</h3>
    <p className="text-muted-foreground">Content in dark mode</p>
    <button className="bg-primary text-primary-foreground px-4 py-2 rounded-md">Primary Action</button>
  </div>
</div>
```

## Usage Guidelines

### Text Colors

- Use `text-foreground` for primary text
- Use `text-muted-foreground` for secondary, less important text
- Use `text-primary` for emphasized text
- Ensure proper contrast ratios (4.5:1 minimum) for all text

### Background Colors

- Use `bg-background` for page backgrounds
- Use `bg-card` for card and container backgrounds
- Use `bg-muted` for subtle background emphasis
- Use `bg-accent` for highlighting without overwhelming

### Interactive Elements

- Use `bg-primary text-primary-foreground` for primary buttons and CTAs
- Use `bg-secondary text-secondary-foreground` for secondary actions
- Use `bg-destructive text-destructive-foreground` for destructive actions
- Use `ring` for focus states

### Accessibility Considerations

- Our OKLCH color system provides better perceptual uniformity
- All color combinations maintain appropriate contrast ratios
- Don't rely solely on color to convey information
- Test in both light and dark modes

